# acceptance/

存放各任務的凍結驗收清單。

- 命名: `<YYYYMMDD>-<任務簡述>.md`（例: `20260703-visitor-reconnect-push.md`）
- 產生時機: 開發流程步驟 0，由 PM 依使用者【原始需求】撰寫清單內容，經使用者確認後凍結
- 寫檔分工: PM 沒有寫檔工具，只在回報中【產出清單內容】；由主 Claude 將內容寫入本目錄的檔案
- 凍結規則: 開發期間（步驟 1~5）任何 agent 不得修改本目錄下已凍結的檔案
- 用途: 步驟 2 與步驟 4 的 PM 驗收【唯一依據】。architect 更新的規格書僅作為文件，不作為驗收依據
- 需求中途變更: 由使用者明示後，回到步驟 0 重新產生並確認新清單，舊清單標記為 superseded

> 註：本目錄在公開範本 repo 中只放本 README 當說明；實際任務清單寫在使用者本機的
> `~/.claude/acceptance/`（不進版控，避免洩漏產品細節）。
